# Exit Ticket Patterns

How the skill generates formative exit tickets tied to the day's success
criteria.

### What an exit ticket does

- Fast formative check (2–5 minutes of student time)
- Tied to the day's success criteria — never to material not covered
- Actionable for the teacher: misconception surfaces before tomorrow

### Common formats

1. **3-2-1** — 3 things learned, 2 questions, 1 connection
2. **Claim-Evidence-Reasoning** — one sentence each
3. **Single-problem mastery check** — one problem students must solve
4. **Fist-to-five self-assessment** — with an open-response "what would make this a 5?"
5. **Muddiest point** — what's still unclear

### Output formats

- Printable half-page (.docx, 2-up per page)
- Google Form template (v0.2+)
- LMS quiz import format: QTI or CSV (v0.2+)

### Anti-patterns to avoid

- Exit tickets that require reading a long passage (not "fast")
- Exit tickets that grade rather than inform
- Exit tickets that ask about content not taught that day
