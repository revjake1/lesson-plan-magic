# Sub Plan Patterns

How the skill converts a regular lesson plan into a guest-teacher-ready
sub plan.

### What a sub plan needs that a regular plan doesn't

- Class rosters (abstract — NOT named — "22 students, 2 with extended-time")
- Known classroom management norms ("students use restroom pass, one at a time")
- Location of materials ("worksheets on top of the gray filing cabinet")
- Technology login notes (without passwords)
- Emergency protocols reference
- Names and rooms of nearby teachers for help
- How to report completion back to the teacher

### What a sub plan must NOT include

- Named students (roster, behavior, accommodation specifics)
- Grades or gradebook access
- Student PII of any kind
- Passwords, credentials, or login QR codes

### Tone adjustment

- Sub plans should be MORE explicit, MORE chunked, MORE idiot-proof than
  regular plans
- No teacher jargon; no framework abbreviations (spell out "SWIRL" if used)
- Estimated times should be generous

### Template pattern

1. Welcome + today's class info
2. Where to find stuff (physical + digital)
3. Period-by-period schedule with activities
4. "If you finish early" backup plan
5. "If there's a problem" escalation path (office phone, nearby teacher)
6. Return-to-teacher note (leave worksheets here, email a quick note about how it went)
