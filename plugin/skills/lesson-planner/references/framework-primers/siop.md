# SIOP — Sheltered Instruction Observation Protocol

For content instruction that's simultaneously accessible to English learners. Echevarría, Vogt, Short.

## Core components

SIOP has 8 components (30 features). For runtime use, emphasize:

1. **Lesson Preparation** — every lesson has BOTH a content objective and a language objective, posted and referenced.
2. **Building Background** — connect to students' experience and prior learning; pre-teach key vocabulary.
3. **Comprehensible Input** — speech rate and vocabulary appropriate to proficiency; use visuals, gestures, modeling.
4. **Strategies** — scaffold higher-order thinking; teach explicit learning strategies.
5. **Interaction** — frequent opportunities for students to use English with peers and teacher; wait time.
6. **Practice & Application** — hands-on activities linking language and content.
7. **Lesson Delivery** — objectives supported; pacing appropriate; students engaged 90%+ of time.
8. **Review & Assessment** — review key vocabulary and concepts; provide feedback; multiple assessment formats.

## How it shapes a lesson plan

- A SIOP-aligned plan shows a **language objective** next to every **content objective**
- Vocabulary is identified in advance, not on-the-fly
- Interaction structures are named explicitly (not just "discussion")
- Assessment checks both content mastery AND language use

## Language signals

- "Content objective: … Language objective: …"
- "Tier 2 academic vocabulary: [words]"
- "Sentence frame: I think ___ because ___"
- "Student talk time ≥ 50%"

## Common pitfalls

- Treating SIOP as an ELL-only add-on; it's a design protocol for the whole class
- Omitting the language objective when the content is "easy"
- Posting objectives without actually referencing them mid-lesson
