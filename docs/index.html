Lesson Planner

Weekly, daily, or unit plans — filled into your district's template.

  • Real standard codes from your uploaded state standards — never fabricated
  • Learning intentions and success criteria on every day
  • Framework breakdown: 5E, gradual release, workshop, SIOP, and more
  • Differentiation for ELL, IEP/504, gifted, and tiered readiness — by population, never by name

Classroom Artifacts

All the short, printable things you need every day — pulled from your plan automatically.

Getting started

Two ways to run it

Both produce identical plans and artifacts. The difference is only in how you install and interact.

★ Recommended

Cowork

A purpose-built desktop app with a drag-and-drop plugin installer

  • ✓ Drag-and-drop .plugin install — click Accept and you're done
  • ✓ Type /lesson-planner to trigger instantly; type / to browse all skills
  • ✓ Upload an exported .ics or selectable-text PDF calendar so holidays skip automatically
  • ✓ Google Forms-ready exit ticket text is built in
  • ✓ Cleaner, more approachable interface — designed for knowledge workers

Claude Code

The technical alternative if you're already using it

  • ✓ Install via Settings → Plugins
  • ✓ Just talk in plain English — no slash command needed
  • ✓ Upload .ics or selectable-text PDF calendar file manually
  • ✓ Good if your IT department has already set it up for you

Both require Python 3.9+ (one-time setup) and a claude.ai account.

Installation — Cowork path

Up and running in 5 steps

No command-line experience needed. Python is the only prerequisite, and step 3 takes care of the rest.

1

Download and install Cowork

Get the free desktop app from claude.ai/cowork and install it like any Mac or Windows application.

2

Install the plugin

Locate jakes-lesson-plan-magic.plugin (it came with this package). Drag and drop it into the Cowork window, then click Accept. Done — two new skills appear.

3

Make sure Python is installed

Lesson Plan Magic needs Python 3.9+, but it now installs its own pinned helper packages automatically on first use. No manual pip install step is required for normal setup.

That first helper install may take a minute or two and needs normal internet access.

If that automatic install fails, the plugin shows plain-English next steps instead of a developer traceback.

4

Add your school calendar (optional but great)

Export an .ics file from Google Calendar or Outlook, or upload a PDF with selectable text from your district website. Upload it during setup, or later with Update my config. Once loaded, the plugin can skip breaks and account for half-days or testing windows when sequencing plans. Scanned-image PDFs usually will not parse reliably.

5

Type /lesson-planner and follow the setup

The first run is a 10–20 minute guided conversation — your name, subjects, standards, district template, and a folder of past plans (optional, but this is what makes it sound like you). Progress saves automatically, so interruptions are fine. You only do this once.

Say what you need. Get a finished document.

The plugin asks 2–3 clarifying questions if needed, then runs each day in parallel so a full week doesn't take five times as long as a single day.

Example requests

Use these exactly or adapt freely:

  • Plan the next three weeks of American Lit, ending with a Socratic seminar.
  • I need a polished daily plan for Thursday — my principal is observing.
  • Plan my week. (elementary — covers all content areas)
  • We had a fire drill. Re-plan Friday's lesson.
  • Sub plan for tomorrow — I'm out sick.

Privacy & student data

Four guarantees. Hard-coded. Non-negotiable.

These aren't settings. They can't be turned off, overridden, or changed by an administrator.

✓

Never writes student names

Differentiation always uses population language — "students with extended-time accommodations" — never a student's name. FERPA-safe by design.

✓

PII scan before every save

Every file is scanned for names, SSNs, phone numbers, email addresses, student IDs, medical notes, and IEP phrasing before it's written to disk. If something is found, the file isn't saved.

✓

No plugin telemetry or hosted storage

Generated files stay local and privacy.telemetry: off is hard-coded. If you use verified links or standards URLs, the plugin makes normal outbound web requests from your machine to fetch or verify that content.

✓

Never fabricates citations

Every link in a plan is verified as live and accurate before inclusion. If a resource can't be confirmed, it's described in plain English — never written as a fake link.

FAQ

Common questions

Do I need a Claude.ai account?
Yes — both Cowork and Claude Code run on Anthropic's Claude AI. You'll need a free or paid account at claude.ai.
What if I don't have my district's lesson plan template?
Choose one of three built-in starters during onboarding: Weekly block (90-min periods), Weekly bell (5–7 period schedule), or Daily one-pager. You can always swap in your real template later by saying I have a new district template to upload.
What if I don't have my state standards?
The plugin still works — it skips the standard-code step and flags that in the compliance notes. You can add standards at any time by saying upload new standards for [subject] and dropping a PDF or pasting a URL.
Can elementary teachers use this?
Yes. Self-contained classrooms are fully supported. You set up one "subject" (e.g., "3rd Grade") with content areas inside — math, reading, writing, science, social studies. The plugin generates one cohesive plan for your whole day.
Can I use this for co-taught classes?
Yes. During onboarding, say the class is co-taught and give your co-teacher's name and role (special education, ELL support, or content specialist). The differentiation sections will automatically include suggestions for splitting the instructional work between you.
The plan doesn't sound like me. What do I do?
First, check that you uploaded past plans during onboarding — voice matching gets dramatically better with even 5–10 old Word documents. If it still sounds off, say The output doesn't sound like me — can you adjust the voice? or Set my voice match level to strict.
I got interrupted during onboarding. How do I resume?
Progress is saved automatically after every step. Just say Resume my setup and it picks up where you left off.
Can it post directly to Google Classroom or Canvas?
Not in this version — LMS integration is on the roadmap. For now, open the output file and upload manually. It takes about 10 seconds.
What's the difference between Cowork and Claude Code — really?
They both run the same plugin and produce identical output. Cowork is a cleaner, more approachable desktop app with a drag-and-drop plugin installer and a /skills menu. Claude Code is a more technical tool aimed at developers. If you're not sure which to use, choose Cowork.
Is my students' data safe?
Yes. Every file is scanned for PII before it's saved, and the plugin never writes a student's name into any output. Generated files stay local by default, with no plugin telemetry. If you use verified links or standards URLs, the plugin makes normal outbound web requests from your machine to fetch or verify that content. See the Privacy section for the full list of guarantees.
